# eject tests to scopes

## .what

move all tests from `withSimpleCache.test.ts` and `withSimpleCacheAsync.test.ts` into scoped test files.

## .why

- main test files are large and hard to navigate
- scoped files group related tests by feature
- easier to find tests for specific behavior
- symmetric with extant `scope=meta`, `scope=serde`, `scope=expiration`, etc.

## .scope

| current location | target scope |
|------------------|--------------|
| `describe('asynchronous logic, synchronous cache')` | `scope=async-logic` |
| `describe('asynchronous logic, asynchronous cache')` | `scope=async-cache` |
| `describe('cache from input')` | `scope=cache-from-input` |
| `describe('expiration')` | already `scope=expiration` |
| `describe('bypass')` | already `scope=bypass` |
| `describe('meta option')` | already `scope=meta` |
| `describe('cache type specificity')` | already `scope=serde` |

## .outcome

main `.test.ts` files become thin or empty, all tests live in `scope=*.test.ts` files.
