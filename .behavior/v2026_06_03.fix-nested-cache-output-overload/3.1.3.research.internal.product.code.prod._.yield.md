## pattern.3 = dynamic cache extraction [REUSE]

the `getCacheFromCacheChoice` function resolves cache at runtime from function args.

**file**: `src/domain.operations/options/getCacheFromCacheChoice.ts:9-47`

```ts
export type SimpleCacheExtractionMethod<
  LI extends any[],
  C extends SimpleCache<any>,
> = (args: { fromInput: LI }) => C;

export type WithSimpleCacheChoice<
  LI extends any[],
  C extends SimpleCache<any>,
> = C | SimpleCacheExtractionMethod<LI, C>;

export const getCacheFromCacheChoice = <LI extends any[], C extends SimpleCache<any>>({
  forInput,
  cacheOption,
}: {
  forInput: LI;
  cacheOption: WithSimpleCacheChoice<LI, C>;
}): C => {
  if (isAFunction(cacheOption)) {
    const foundCache = cacheOption({ fromInput: forInput });
    if (!foundCache) throw new BadRequestError(...);
    return foundCache;
  }
  return cacheOption;
};
```

**relation to wish**: this pattern already handles dynamic cache extraction. when `meta: 'include'`, we will use this same pattern to get the cache, then access `cache.uri` if available.

**cite [3]**: lines 9-12 define `SimpleCacheExtractionMethod`, lines 19-22 define `WithSimpleCacheChoice`, lines 27-47 define `getCacheFromCacheChoice`.

## pattern.4 = SimpleCache interface [EXTEND]

the cache interface defines get/set methods but no uri property.

**file**: `src/domain.objects/SimpleCache.ts:6-30`

```ts
export interface SimpleSyncCache<T> {
  get: (key: string) => T | undefined;
  set: (key: string, value: T | undefined, options?: { expiration?: UniDuration | null }) => void;
}

export interface SimpleAsyncCache<T> {
  get: (key: string) => Promise<T | undefined>;
  set: (key: string, value: T | undefined, options?: { expiration?: UniDuration | null }) => Promise<void>;
}

export type SimpleCache<T> = SimpleAsyncCache<T> | SimpleSyncCache<T>;
```

**relation to wish**: we need to extend this to optionally include `uri`. options:
1. add optional `uri?: string` property to interfaces
2. add optional `getUri?: (key: string) => string` method
3. duck-type check `'uri' in cache` at runtime

option 3 is safest for backwards compat — cache implementations that support uri can add the property without defects to extant caches.

**cite [4]**: lines 6-13 define `SimpleSyncCache`, lines 18-25 define `SimpleAsyncCache`, line 30 defines `SimpleCache`.
