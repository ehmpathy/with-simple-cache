# internal research: test codepaths

## pattern.1 = test fixture cache factories [EXTEND]

test fixtures create sync and async caches with exposed stores for inspection.

**file**: `src/.test.assets/createExampleCache.ts:5-39`

```ts
export const createExampleSyncCache = () => {
  const store: Record<string, any> = {};
  const cache: SimpleSyncCache<any> = {
    set: (key: string, value: any, options?: { expiration?: UniDuration | null }) => {
      store[key] = options ? { value, options } : { value };
    },
    get: (key: string) => store[key]?.value,
  };
  return { cache, store };
};

export const createExampleAsyncCache = <T>() => {
  const store: Record<string, { value: T; options?: any } | undefined> = {};
  const cache: SimpleAsyncCache<T> = {
    set: async (key: string, value: T | undefined, options?: ...) => {
      store[key] = value !== undefined ? (options ? { value, options } : { value }) : undefined;
    },
    get: async (key: string) => store[key]?.value,
  };
  return { cache, store };
};
```

**relation to wish**: need to extend with `createExampleAsyncCacheWithUri` that includes `uri` property. the test fixture exposes both cache and store, which allows tests to verify what was cached.

**cite [1]**: lines 5-18 define `createExampleSyncCache`, lines 20-39 define `createExampleAsyncCache`.

## pattern.7 = error exposure test [REUSE]

tests verify errors propagate correctly through the cache wrapper.

**file**: `src/domain.operations/wrappers/withSimpleCacheAsync.test.ts:64-81`

```ts
it('should expose the error, if an error was rejected by the promise returned by the function', async () => {
  const expectedError = new Error('surprise!');
  const callApi = withSimpleCacheAsync(
    async () => {
      throw expectedError;
    },
    { cache: createExampleSyncCache().cache },
  );

  try {
    await callApi();
    throw new Error('should not reach here');
  } catch (error) {
    expect(error).toEqual(expectedError);
  }
});
```

**relation to wish**: error tests will verify errors still propagate when `meta: 'include'`.
