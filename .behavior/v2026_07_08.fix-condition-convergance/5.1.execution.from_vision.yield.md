## the change, in brief

- **predicate** (`isSimpleCacheConditionError.ts`): three name-based checks, no `instanceof` against our own classes:
  1. `error instanceof Error` — the global builtin (cross-copy safe) as the floor
  2. `error.constructor?.name === 'SimpleCacheConditionError'` — leaf discriminator; a realm-global string that matches regardless of which package minted the class
  3. a ConstraintError ancestor *by name* (`hasProtoConstructorNamed`, walks the prototype chain) — narrows the blast radius so an unrelated Error that merely reuses the leaf name is not swallowed on the `exception: ignore` path. name-based, so it too survives a duplicate helpful-errors copy.
- **both guards**: `if (!(conditionOption && isSimpleCacheConditionError(error))) throw error;` — identical in sync + async per `rule.require.wrapper-symmetry`.
- **test fidelity**: the cross-package case is simulated in-process via a `ForeignSimpleCacheConditionError extends ConstraintError` whose `name` is overridden to `'SimpleCacheConditionError'` and whose constructor differs from ours. the tests assert `error instanceof SimpleCacheConditionError === false` (so instanceof would miss it) AND that convergence still happens (so the name check catches it).

### design reconciled vs vision Q4 (recorded so the yield is the source of truth)

the vision Q4 left the discriminator *lane* open for the wisher. what shipped:
- **name-based, no instanceof against our classes** — per vlad's explicit "check on the name alone". this is looser than vision Q4's option-1 (which paired the name with `instanceof ConstraintError`), *on purpose*: `instanceof ConstraintError` shares the exact cross-copy fragility the fix exists to defeat.
- **but re-tightened by a name-based ConstraintError ancestry walk** (added in response to l3 peer r11) — recovers option-1's blast-radius safety *without* a return of the cross-copy `instanceof` weakness. so the shipped design is: name-based leaf discriminator + name-based ancestry guard. strictly better than every vision Q4 lane except the branded-marker, which is the deferred handoff.
- **magic string kept a literal, deliberately** (vision Q1 leaned "derive from the class"): a *derived* `SimpleCacheConditionError.name` would follow *our* rename and stop to match conformant foreign backends (whose class keeps the old name), and would mangle first under our own minification. the literal is the correct cross-package wire string. extracted to a named const (`nameOfConditionError`) with the rationale inline, so it is a documented contract constant, not a bare magic value.
